# HSW Documentation Index

This directory is the durable documentation home for Hoshi Reader Windows.

Use these files before falling back to the root `STATUS.md`:

- `VALIDATION.md` - durable validation commands, focused probes, runtime checks, and packaging rules.
- `ARCHITECTURE.md` - current architecture facts, module boundaries, and data-flow ownership.
- `ARCHITECTURE_REFACTORING.md` - future architecture debt and refactor direction.
- `PROJECT_STATUS.md` - current implementation status, known gaps, not-verified items, and risks.
- `TODO.md` - short operational handoff for the next agent or slice.
- `LOOKUP_ROADMAP.md` - remaining reader lookup slices; execute only one slice at a time.
- `REAL_LOOKUP_VALIDATION_PLAN.md` - real hoshidicts toolchain, dictionary import, and runtime lookup validation slices.
- `ANKI_HSA_WINDOWS_ALIGNMENT.md` - HSA/Windows Anki integration evaluation and follow-up slices.
- `READER_ENTRY_CHARACTERIZATION.md` - bookshelf/import/reader/session behavior baseline.
- `reader-layout-baseline.md` - vertical reader pagination and layout measurement baseline.

The root `STATUS.md` is now only a short pointer to these docs. If content would overlap, keep the detailed version in `docs/`.
